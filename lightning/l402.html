L402: pagos nativos en la web y APIs monetizadas

El código HTTP 402 "Payment Required" llevaba 30 años sin uso real. L402 le da vida con Lightning.

El código que nadie usaba

HTTP 402 existía en la especificación, pero sin una infraestructura de micropagos viable no tenía uso real. Con Lightning y Macaroons, L402 convierte ese código en un estándar práctico para acceso de pago a recursos web y APIs.

Cómo funciona L402

  1. Cliente solicita recurso protegido.
  2. Servidor responde 402 con invoice Lightning + macaroon.
  3. Cliente paga invoice y obtiene preimage.
  4. Cliente reintenta con macaroon + prueba de pago.
  5. Servidor valida criptográficamente y sirve recurso.

Por qué es distinto a modelos tradicionales

  • Pago = autenticación: no necesitas cuenta ni sesión clásica.
  • Micropagos reales: precios por petición en sats.
  • Pay-per-use puro: pagas lo que consumes.
  • Machine-native: agentes pueden operar sin tarjeta ni KYC.

Aperture y despliegue práctico

Aperture (reverse proxy) permite poner L402 delante de servicios HTTP existentes sin reescribir el backend de negocio. El proxy gestiona negociación 402, validación y forwarding.

Macaroons en L402

Los macaroons aportan permisos granulares y verificación stateless. Permiten restringir alcance temporal, capacidades y presupuesto sin depender de sesiones tradicionales.

IA y comercio máquina-a-máquina

L402 es una pieza clave para que agentes de IA paguen APIs autónomamente usando sats como credencial económica nativa.
