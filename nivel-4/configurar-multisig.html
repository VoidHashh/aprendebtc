Configurar multisig con Sparrow Wallet

Guía práctica para crear tu primera wallet multisig 2-de-3 usando Sparrow y tus hardware wallets.

⚠️
Versión y cambios de interfaz.

Esta guía usa Sparrow Wallet en su versión disponible a marzo de 2026. La interfaz puede variar en versiones futuras. Consulta siempre sparrowwallet.com/docs para la documentación más actualizada antes de proceder.

Antes de empezar: qué necesitas

  • 2 o 3 hardware wallets: pueden ser del mismo modelo o de fabricantes distintos. Usar dispositivos de diferentes fabricantes (por ejemplo, una Coldcard y una Trezor) añade diversificación: si se descubre una vulnerabilidad en el firmware de uno, el otro no se ve afectado.
  • Sparrow Wallet instalada: descárgala desde sparrowwallet.com. Verifica siempre la firma del instalador antes de ejecutarlo.
  • Conexión al servidor: Sparrow puede conectarse a tu propio nodo Bitcoin (recomendado para máxima privacidad) o a un servidor público. Para esta guía, cualquiera de las dos opciones funciona.
  • Tiempo y tranquilidad: no hagas esto con prisa. Reserva al menos una hora sin interrupciones para el setup inicial y las verificaciones.
⚠️
Lee primero.

Antes de seguir esta guía, asegúrate de haber leído ¿Qué es multisig? para entender los conceptos. Si tienes dudas sobre qué es una seed phrase o un hardware wallet, visita primero Configurar tu hardware wallet.

Paso 1: Crear la nueva wallet en Sparrow

  1. Abre Sparrow Wallet.
  2. En el menú superior: File → New Wallet.
  3. Elige un nombre descriptivo para tu wallet. Algo como multisig-ahorro-2de3 ayuda a identificarla claramente si tienes varias wallets en Sparrow.
  4. En la pantalla de configuración, localiza el campo Policy Type y selecciona Multi Signature.
  5. Configura el esquema M-de-N: en este caso, 2 Required (M) y 3 Cosigners (N).

Paso 2: Configurar los tres cosignatarios (keystores)

Sparrow te mostrará tres slots de cosignatario (Keystore 1, Keystore 2, Keystore 3). Para cada uno, debes importar la clave pública extendida (xpub) del dispositivo correspondiente.

Keystore 1: primer hardware wallet

  1. Conecta tu primer hardware wallet al ordenador.
  2. En el slot Keystore 1, haz clic en Connected Hardware Wallet.
  3. Sparrow detectará el dispositivo. Selecciónalo y haz clic en Import Keystore.
  4. Confirma la operación en el dispositivo si te lo pide (en la pantalla del hardware wallet).
  5. Sparrow importará la xpub de ese dispositivo. Verás el fingerprint (huella) de la clave: anótalo para futuras referencias.

Keystore 2: segundo hardware wallet

  1. Desconecta el primero, conecta el segundo hardware wallet.
  2. Repite el mismo proceso en el slot Keystore 2.

Keystore 3: tercera clave

Tienes dos opciones para la tercera clave:

  • Opción A: un tercer hardware wallet, igual que los anteriores. La opción más robusta.
  • Opción B: un backup de seed en papel o metal, del que derivas la xpub usando Sparrow o una herramienta offline. Esta opción es más arriesgada porque la clave no está en un dispositivo de seguridad dedicado, pero puede ser válida si la ubicación física es muy segura (por ejemplo, una caja fuerte en otra ciudad).

En ambos casos, el procedimiento de importación en Sparrow es el mismo.

Paso 3: Seleccionar el tipo de script

En el campo Script Type, selecciona Native Segwit (P2WSH). Es el estándar moderno para multisig: genera las direcciones de menor coste en fees y es compatible con la gran mayoría de hardware wallets actuales.

Evita P2SH (el tipo de script más antiguo) a menos que tengas un motivo específico de compatibilidad.

Paso 4: Aplicar y verificar la wallet

  1. Haz clic en Apply. Sparrow te pedirá una contraseña opcional para cifrar el archivo de wallet en tu ordenador, recomendable.
  2. La wallet está creada. Verás la pantalla principal con la dirección de recepción.

Verificación crítica: confirmar la dirección en los dispositivos

Este paso es obligatorio antes de enviar fondos reales. Debes confirmar que la dirección de recepción que muestra Sparrow coincide con la que muestran al menos 2 de tus 3 hardware wallets cuando les pides verificar la dirección.

  1. En Sparrow, ve a la pestaña Receive y copia la primera dirección de recepción.
  2. Conecta cada hardware wallet y usa su función de verificación de dirección (en Coldcard: Address Explorer; en Trezor/Ledger: Verify address en Sparrow con el dispositivo conectado).
  3. Confirma que la dirección en el dispositivo coincide exactamente con la que muestra Sparrow. Si no coincide, detente y no envíes fondos.
🚨
No omitas la verificación de dirección.

Un malware en tu ordenador podría modificar la dirección que muestra Sparrow. La única forma de saber que la dirección es realmente la tuya es confirmarla directamente en la pantalla de tus hardware wallets.

Paso 5: Prueba con fondos pequeños

Antes de mover cantidades importantes, haz una prueba completa:

  1. Envía una cantidad pequeña (el mínimo razonable dado las fees actuales) a la dirección que acabas de verificar.
  2. Espera a que se confirme.
  3. Crea una transacción de prueba para devolverte esos fondos a otra wallet tuya.
  4. Firma esa transacción con exactamente 2 de tus 3 hardware wallets para confirmar que el flujo de firma funciona correctamente.

Solo cuando hayas completado este ciclo completo (recibir + enviar + confirmar), la configuración está verificada.

Paso 6: El backup del descriptor, tan importante como las seeds

En una wallet multisig, además de las seeds de cada dispositivo, existe el wallet descriptor: un archivo que describe la estructura completa de la wallet (qué claves la forman, en qué orden, con qué tipo de script). Sin él, aunque tengas las 3 seeds, no puedes reconstruir la wallet.

🚨
El descriptor es tan importante como las seeds.

Con las 3 seeds pero sin el descriptor, no puedes reconstruir tu wallet multisig. Guarda siempre una copia del descriptor junto con cada copia de cada seed, en cada ubicación física donde guardes material de backup.

Cómo exportar el descriptor en Sparrow

  1. Con la wallet abierta en Sparrow, ve a File → Export Wallet.
  2. Elige el formato Output Descriptor (o el formato específico compatible con la herramienta de recuperación que planeas usar).
  3. Guarda el archivo en múltiples ubicaciones: junto con cada seed, en una memoria USB separada, impreso en papel.

Cómo firmar transacciones: el flujo PSBT

Una vez que la wallet está configurada, cada vez que quieras enviar bitcoin necesitarás firmar con al menos 2 de los 3 dispositivos. Sparrow usa el estándar PSBT (Partially Signed Bitcoin Transaction) para gestionar este proceso:

  1. Crea la transacción en Sparrow (monto, destino, fee).
  2. Conecta el primer hardware wallet. Sparrow le envía la transacción sin firmar. El dispositivo muestra los detalles: verifica en la pantalla del dispositivo que la dirección de destino y el monto son correctos. Firma.
  3. Conecta el segundo hardware wallet. Repite el proceso. Con 2 firmas, la transacción está lista para emitir.
  4. Sparrow difunde la transacción a la red.
ℹ️
Por qué Sparrow para multisig.

Sparrow es la wallet de escritorio más recomendada para multisig por su interfaz clara, su soporte completo de PSBT, y su compatibilidad con los principales hardware wallets del mercado. También es software de código abierto, auditable por cualquiera.

El siguiente paso: distribuir correctamente las claves

La configuración técnica es solo la mitad del trabajo. La otra mitad es decidir dónde y cómo guardar físicamente las 3 claves (y el descriptor) para que sean al mismo tiempo seguras contra pérdida, robo y desastre físico.

⚠️
No improvises la distribución.

Antes de guardar las seeds en ubicaciones definitivas, lee el artículo de distribución geográfica de claves. Una mala distribución puede crear vulnerabilidades que anulan toda la seguridad de multisig.
